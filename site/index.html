  1. 📋 Brief 관리 탭 → + 새 Brief 클릭
  2. 양식 작성 (모든 항목 필수)
  3. 저장 → Draft 상태로 등록

Brief 상태 흐름

Draft → Live → ✅ 유효 ❌ 기각 ⏳ 미확정
상태색상언제 바꾸나
Draft🔘 회색Brief 처음 작성 시
Live🔵 파랑소재가 실제로 게시/집행되면
✅ 유효🟢 초록검증 기준 충족 시
❌ 기각🔴 빨강검증 기준 미달 시
⏳ 미확정🟡 노랑데이터 부족 / 기간 미도래

상태 변경 & 검증 결과 기록

  1. Brief 카드 클릭 → 상세 모달
  2. 상단 드롭다운에서 새 상태 선택 → 즉시 반영
  3. Verified 상태로 바꾸면 검증 결과 기록 입력란 등장
  4. 왜 유효/기각인지 기록 → 결과 저장
💡 검증 결과를 남겨야 다음 Brief 작성 시 참고할 수 있습니다.

검색 & 필터

검색창: 제목, 페르소나, JTBD, 가설에서 키워드 검색
필터 버튼: 전체 / Draft / Live / Verified 상태별 필터링

JSON 내보내기 / 가져오기

Brief 데이터는 브라우저 로컬 저장소에 저장됩니다.

버튼기능
JSON 내보내기모든 Brief를 .json 파일로 다운로드 (백업)
JSON 가져오기.json 파일 선택하여 Brief 일괄 불러오기
⚠️ 가져오기 시 기존 데이터가 덮어쓰기됩니다! 먼저 내보내기로 백업하세요.

주간 리뷰 탭 보는 법

  1. 주차 선택 — 상단 버튼으로 이동
  2. KPI 카드 — 🟢 개선 / 🔴 악화 (CPC 등 비용은 반대)
  3. 섹션별 상세 — 채널 성과, 퍼널, 구매자, 인사이트
💡 데이터는 자동 업데이트됩니다. 마케터가 직접 할 일은 없습니다.

새 소재 기획 시 체크리스트

  • Brief 작성 (모든 필수 필드)
