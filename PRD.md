## 1. 배경

### 문제 정의
2026.03.31 미팅 분석(협업 구조 진단 보고서)에서 도출된 핵심 문제:
- 가설→실험→학습 사이클이 부재한 상태에서 리더와 마케터가 서로 다른 층위의 소통을 하고 있었음
- 같은 질문의 반복 + 논점 이탈이 서로를 강화하는 악순환
- **해법:** Creative Brief 1장 도입으로 소통 비효율 70% 해소 + 구조적 학습 시스템

### 기존 대시보드와의 차이

| 항목 | 기존 (rf-marketing-dashboard) | 신규 (marketing-ops) |
|------|-------------------------------|---------------------|
| 목적 | 할 일 현황 칸반 보드 | 가설→실험→학습 워크플로우 |
| 핵심 기능 | 태스크 관리, 상태 추적 | 주간 리뷰, 구매자 역추적, Brief 관리 |
| 데이터 | tasks.json (수동) | GA4/Meta/네이버/아임웹 자동 수집 |
| 의사결정 지원 | 없음 | 가설 검증, 채널 효율 비교, 퍼널 분석 |

기존 칸반은 "뭘 해야 하는지" → 신규는 "왜 하는지 + 결과가 어땠는지".

## 2. 목표

### 핵심 목표
1. **주간 리뷰 자동화** — 데이터 수집→리포트→가설 검증을 대시보드가 처리
2. **구매자 역추적** — 아임웹 구매→GA4 유입채널→Meta/네이버→체험DB 연결
3. **Creative Brief 등록/검증** — 마케팅 루프 스키마(9요소)와 연동
4. **최종: AI 에이전틱 워크플로우** — 이상 감지→가설 제안→Brief 초안→자동 리포팅

### 비목표 (Scope Out)
- 기존 칸반 대시보드 대체 (별도 유지)
- 마케팅 외 영역 (SW/HW/운영)
- CRM/고객 관리 시스템
- 광고 자동 집행/자동 입찰

## 4. 기능 구조 (탭 기반)

### 탭 1: 주간 리뷰 (Weekly Review)

운영 가이드의 4부 구조를 그대로 구현:

**0부: 구매자 역추적**
- 이번 주 구매 건별: 유입 채널 / 방문 횟수 / 경유 페이지 / 체험 여부
- 누적 패턴: 구매자 페르소나별 비중 변화
- 데이터: 아임웹 주문 + GA4 User Explorer (가능 범위)

**1부: 채널별 성과**
- Meta Ads: 캠페인별 Spend/CTR/CPC/전환, 소재별 피로도
- GA4: 세션/유입채널/전환퍼널/이탈구간
- 아임웹: 실주문 vs GA4 전환 대조
- 인스타그램: Reach/Saves/Shares
- 네이버 SA: 그룹별 CTR/CPC/전환
- B2B 리드: 신규/발송/응답/미팅 전환

**2부: 가설 검증**
- 지난주 Brief에서 세운 가설 → 결과 자동 매칭
- 판정: 유효/기각/미확정 + 근거

**3부: 이번 주 계획**
- 인사이트 기반 실험 가설 + Brief 링크

**4부: 이슈/블로커**
- 리소스 병목, 소재 소진, 기술 이슈

### 탭 2: 구매자 역추적 (Buyer Journey)

- 구매 건별 타임라인 (첫 유입→재방문→구매)
- 채널 어트리뷰션 시각화
- 체험 신청→구매 전환 추적
- 퍼널 단계별 이탈 분석 (view_item→checkout→purchase)

### 탭 3: Brief 관리 (Creative Briefs)

운영 가이드의 마케팅 루프 스키마 9요소 연동:

**Brief 양식:**
- 페르소나 (스키마 ⑤)
- JTBD (스키마 ①)
- VP (스키마 ⑦)
- 소재 구성 (스키마 ⑧)
- 채널 (스키마 ⑨)
- 가설 — "이 조합이 기존 대비 [지표]를 개선할 것이다"
- 검증 기준 — 어떤 지표로, 언제 판단할 것인가

**기능:**
- Brief CRUD (생성/조회/수정/삭제)
- 라이브 후 자동 데이터 연결 → 가설 검증 상태 업데이트
- Brief 히스토리 (어떤 Brief가 효과 있었는지 누적)

### 탭 4: 8주 회고 / 종합 분석

- 이미 생성된 8week_review.md 기반
- 향후 분기별 종합 분석 자동 생성
- 학습 아카이브 (효과 있었던 것 / 없었던 것)
