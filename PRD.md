---

## 1. 핵심 목적

> **시스템이 레일이 된다.**
> 마케터가 가설 기반 업무 방식을 이해했든 안 했든,
> 이 대시보드의 플로우를 따르기만 하면 자연스럽게
> 가설→실험→검증→학습 사이클이 돌아가는 구조.

설득이나 교육이 아니라, **양식의 빈칸이 행동을 강제한다.**
- Brief 없이는 소재/캠페인 등록 불가
- 가설·검증기준 = 필수 필드 (빈칸이면 다음 단계 못 넘어감)
- 주간 리뷰에서 가설 vs 결과 자동 매칭 (마케터가 리포트 만들 필요 없음)
- 축적된 Brief 히스토리가 "왜 됐고 왜 안 됐는지"를 기록으로 남김

---

## 2. 배경

### 문제 (3/31 미팅 진단)
- 가설→실험→학습 사이클 부재 → 리더와 마케터가 서로 다른 층위의 소통
- 같은 질문 반복 + 논점 이탈 악순환
- 소재별 기획 의도 공유 부재 → 피드백이 "의도가 뭐냐"에서 시작
- 성공/실패의 재현·학습 불가

### 해법
Creative Brief 1장이 소통 비효율의 70%를 구조적으로 해소.
이 대시보드는 Brief를 중심으로 전체 마케팅 워크플로우를 강제하는 시스템.

### 기존 대시보드(rf-marketing-dashboard)와의 관계
- 기존 = "뭘 해야 하는지" (칸반 태스크 관리) → 별도 유지
- 신규 = "왜 하는지 + 결과가 어땠는지" (가설→실험→학습)

## 3. 사용자

| 사용자 | 역할 | 시스템이 강제하는 것 |
|--------|------|---------------------|
| 황성우 (리더) | 주간 리뷰, Brief 협의, 방향 결정 | 데이터가 알아서 정리되어 의사결정에 집중 |
| 마케터 | Brief 작성→소재 제작→성과 확인 | Brief 필수 필드가 "왜 이걸 만드는가"를 매번 명문화시킴 |
| AI (고릴/독술) | 데이터 수집, 리포트 생성, 이상 감지 | 자동 |

## 4. 기능 구조 (4탭)

### 탭 1: 📊 주간 리뷰 (Weekly Review)

노션 운영 가이드의 4부 구조를 그대로 구현. **대시보드가 자동으로 수집·정리하므로 마케터가 직접 데이터를 모으거나 리포트를 만들 필요 없음.**

**0부: 구매자 역추적**
- 이번 주 구매 건별: 유입 채널 / 방문 횟수 / 경유 페이지 / 체험 여부
- 누적 패턴: 구매자 페르소나별 비중 변화

**1부: 채널별 성과**
- Meta Ads: 캠페인별 Spend/CTR/CPC/전환, 소재별 피로도
- GA4: 세션/유입채널/전환퍼널/이탈구간
- 아임웹: 실주문 vs GA4 전환 대조
- 인스타그램: Reach/Saves/Shares
- 네이버 SA: 그룹별 CTR/CPC/전환
- B2B 리드: 신규/발송/응답/미팅 전환

**2부: 가설 검증**
- 지난주 Brief에서 세운 가설 → 결과 자동 매칭
- 판정: ✅ 유효 / ❌ 기각 / ⏳ 미확정 + 근거

**3부: 이번 주 계획**
- 인사이트 기반 실험 가설 + Brief 링크

**4부: 이슈/블로커**
- 리소스 병목, 소재 소진, 기술 이슈, 리더 의사결정 필요 사항

### 탭 2: 🔍 구매자 역추적 (Buyer Journey)

- 구매 건별 타임라인 (첫 유입→재방문→구매)
- 채널 어트리뷰션 시각화
- 체험 신청→구매 전환 추적
- 퍼널 단계별 이탈 분석 (view_item→checkout→purchase)

### 탭 3: 📋 Brief 관리 (Creative Briefs) — **핵심 탭**

**🔴 Brief = 소재 라이브의 전제 조건**
Brief 없이는 캠페인/소재를 시스템에 등록할 수 없음.

**Brief 양식 (크리에이티브 스키마 9요소 연동):**

| 필드 | 스키마 연동 | 필수 |
|------|-----------|------|
| 제목 | 캠페인/소재명 | ✅ |
| 페르소나 | ⑤ PersonaSeed + WHO | ✅ |
| JTBD | ① Problem + JTBD | ✅ |
| VP | ⑦ Value Proposition | ✅ |
| 소재 구성 | ⑧ Creative Structure | ✅ |
| 채널 | ⑨ Channel | ✅ |
| **가설** | "이 조합이 기존 대비 [지표]를 개선할 것이다" | **🔴 필수** |
| **검증 기준** | 어떤 지표로, 언제 판단할 것인가 | **🔴 필수** |
| 상태 | draft → live → verified | 자동 |

**플로우 (시스템이 강제하는 순서):**
```
Brief 작성 (마케터) — 필수 필드 모두 채워야 저장
  ↓
Brief 협의 (마케터 + 리더) — 여기서 방향 확정
  ↓
상태: live — 제작 완료, 라이브 날짜 입력
  ↓
데이터 자동 수집 — 검증 기준 기간 도래 시 자동 매칭
  ↓
가설 검증 — ✅ 유효 / ❌ 기각 / ⏳ 미확정
  ↓
학습 기록 — Brief 히스토리에 축적
```

**기능:**
- Brief CRUD (생성/조회/수정/삭제)
- 스키마 드롭다운 선택 (직접 타이핑 최소화)
- 라이브 후 자동 데이터 연결 → 가설 검증 상태 업데이트
- Brief 히스토리 (어떤 Brief가 효과 있었는지 누적)
- Brief별 성과 비교 (같은 VP끼리, 같은 페르소나끼리 등)

### 탭 4: 📈 8주 회고 / 종합 분석

- 8week_review.md 기반 (W6~W13 교차검증 완료)
- 분기별 종합 분석 자동 생성
- 학습 아카이브 (효과 있었던 것 / 없었던 것)
- Brief 축적 데이터 기반 패턴 발견
