# 차트 시각화 가이드라인 — 상황별 패턴 사전

> 출처: Storytelling with Data (Cole Nussbaumer Knaflic), Datawrapper 블로그, Data-to-Viz.com, Tufte "The Visual Display of Quantitative Information", Friends Don't Let Friends (Chenxin Li), Dark Horse Analytics

---

## 🔴 절대 하면 안 되는 것 (Anti-patterns)

### 1. 듀얼 Y축
- **문제:** 두 축의 스케일이 자의적 → 교차점이 거짓 상관 유발
- **연구:** 15명 중 14명이 "가장 혼란스러운 차트" (Isenberg 2011)
- **대안:** 스몰 멀티플, 인덱스 차트, 우선 지표 하나만 차트+나머지 주석

### 2. 독립 정규화 멀티라인
- **문제:** 각 시리즈를 자기 min~max로 0~100% 매핑 → 모양이 비슷해 보이면 상관있다고 착각
- **대안:** 인덱스 차트(시작=100, 변화율), 스몰 멀티플

### 3. 스파게티 차트 (5개 이상 라인)
- **문제:** 5개 넘으면 어떤 선이 뭔지 추적 불가
- **대안:** ① 1개 하이라이트 + 나머지 회색 ② 스몰 멀티플 ③ 인터랙티브 호버

### 4. 파이 차트
- **문제:** 인간은 각도 비교를 못 함. 비슷한 비율 구별 불가
- **대안:** 가로 바 차트 (값 순서대로 정렬), 트리맵

### 5. 바 차트에서 Y축 0 아닌 시작
- **바 차트:** 길이=값이므로 반드시 0에서 시작. 안 하면 차이가 과장됨
- **라인 차트:** 0에서 시작 안 해도 됨 (위치 기반이라 변화가 중요)

### 6. 바 차트로 평균 비교 (분포 숨기기)
- **문제:** 바 차트는 평균만 보여줘서 분포 차이를 완전히 숨김
- **대안:** 점+박스플롯, 바이올린, 스웜플롯

### 7. 빨강/초록 색상 조합
- **문제:** 색각 이상자 8% 구분 불가
- **대안:** 파랑/주황, 파랑/빨강 등

## ✅ 상황별 차트 선택 패턴

### 상황 1: 시간에 따른 추이 1~2개 지표
**→ 라인 차트 (단일 Y축)**
- Y축 0에서 시작 안 해도 됨
- 직접 라벨 (범례보다 라인 끝에 이름)
- 영역 채우기는 1개 시리즈만

### 상황 2: 시간 추이 + 스케일이 완전히 다른 2개 지표 비교
**→ 스몰 멀티플 (위아래 쌓기)**
```
┌─ 광고비 (₩) ──────────────┐
│  ₩100만 ──╲──╱──────       │
│  ₩50만  ────╲─────         │
│         W1  W2  W3  W4     │
├─ ROAS (%) ─────────────────┤
│  1000% ──────╱──────       │
│  500%  ────╱───────        │
│         W1  W2  W3  W4     │
└────────────────────────────┘
```
- 같은 X축 공유, Y축 각자
- 시간 정렬이 동일해서 "이 주에 광고비 올렸는데 ROAS도 올랐네" 비교 가능
- 각 차트의 Y축이 독립이라 스케일 혼동 없음

### 상황 3: 여러 지표의 변화율 비교 (스케일 무관하게)
**→ 인덱스 차트 (시작=100)**
- 기준 주차 = 100%, 이후 변화율로 표시
- 광고비 +50%, ROAS +200%, 세션 -10% → 같은 축에서 비교 가능
- **단점:** 절대값 정보 없음 → 호버 툴팁이나 좌측 라벨로 보완
- **주의:** 변화율 차이가 극심하면 (하나가 +10000%, 다른 게 +5%) 작은 것이 평평해짐

### 상황 4: 카테고리별 크기 비교
**→ 가로 바 차트 (값 순서 정렬)**
- 반드시 **값 기준 내림차순 정렬** (알파벳 순 금지)
- Y축 = 0에서 시작
- 카테고리 10개 넘으면 → 상위 N개 + "기타" 묶기
- 예: VP별 소재 수, 캠페인별 CPA

### 상황 5: 절대값(건수) + 비율(%) 동시에 보여줘야 할 때
**→ 바 + 라인 콤보 (단, 스몰 멀티플 우선)**
```
스몰 멀티플이 더 좋지만, 공간이 부족하면:
  바 = 절대값 (좌축, ₩/건)
  라인 = 비율 (우축, %)
```
- 바가 주(primary), 라인이 보조(secondary) — 명확하게 역할 구분
- **우축 사용 시 규칙:** 우축 라벨 색상 = 라인 색상과 동일
- 바-라인 교차점에 의미 부여하지 말 것 (스케일이 다르니까)

### 상황 6: 전체에서 부분 비중
**→ 스택 바 차트 (100% 기준) 또는 트리맵**
- 시간별 비중 변화 → 100% 스택 바
- 한 시점의 구성 → 트리맵 또는 도넛(항목 ≤4개만)
- 항목별 비교가 중요하면 → 일반 바 차트가 나음

### 상황 7: 작은 절대값 (0~10건)
**→ 도트 차트 or 숫자 직접 표기 (라인 차트 금지)**
- 본체 판매 3대 같은 건 선 차트로 그리면 아무 정보가 없음
- 숫자 카드로 보여주고, 추이가 필요하면 점+선으로 점 크게, 직접 라벨

### 상황 8: 캠페인/이벤트 기간 시각화
**→ 간트 차트 (수평 바)**
- 라이브 구간 = 색 채운 바, 중단 구간 = 빈 바 or 점선
- 콘텐츠 업로드 시점 = 이모지 마커 (📸🎬📝)
- 주요 이벤트 = 수직 점선 + 라벨

### 상황 9: 두 변수의 관계 (상관)
**→ 스캐터플롯**
- X = 광고비, Y = 전환 → 상관관계 시각적 확인
- 각 점에 라벨 (캠페인명 등)
- 추세선은 확실한 경우만

### 상황 10: 하이라이트 + 맥락
**→ 1개 강조 + 나머지 회색**
- "이 캠페인이 특별했다"를 보여줄 때
- 메인 시리즈 = 색상 + 두꺼운 선 + 직접 라벨
- 나머지 = 회색 얇은 선 (맥락 제공, 주목 분산 방지)

## 🎨 색상 규칙

### 의미 매핑 (고정)
| 의미 | 색상 | HEX |
|---|---|---|
| 긍정/성장/hit | 초록 | `#10b981` |
| 부정/하락/miss | 빨강 | `#ef4444` |
| 주의/경고 | 주황 | `#f59e0b` |
| 중립/미확정 | 회색 | `#6b7280` |
| Meta 광고 | 파랑 | `#3b82f6` |
| 네이버 광고 | 녹색 | `#22c55e` |
| 오가닉 | 보라 | `#8b5cf6` |

### 시리즈 색상 (구분용, 최대 5개)
```
#6366f1  #f59e0b  #10b981  #ec4899  #3b82f6
```
- 5개 넘으면 스몰 멀티플로 분리
- 배경과 최소 4.5:1 대비 (다크 테마 주의)

## 📝 텍스트/주석 원칙

### 차트 제목 = "답"을 쓴다
- ❌ "주간 세션 추이"
- ✅ "W12 이후 세션 30% 감소 — 네이버 광고 중단 영향"

### 직접 라벨 > 범례
- 라인 끝이나 데이터 포인트 옆에 직접 이름 표기
- 범례는 눈이 왔다갔다해야 해서 인지 부하 큼

### 이벤트 주석
- 수직 점선 + 짧은 라벨: "스포엑스 3/27~30", "캠페인 중단"
- 차트 영역 밖이 아니라 차트 안에 배치

### Data-Ink Ratio (Tufte)
- "지울 수 없는 핵심 잉크"만 남기기
- 불필요한 격자선, 테두리, 장식 제거
- 배경색/그라데이션 제거 (데이터와 무관한 잉크)

## 🏗️ 레이아웃 원칙

### 시선 흐름
- 가장 중요한 차트 = 좌상단
- 한 뷰에 차트 3~4개 이내
- 관련 차트는 수직 정렬 (X축 공유)

### 스몰 멀티플 배치
- 같은 X축/Y축 스케일 공유 → 비교 용이
- 2열이 기본, 모바일은 1열
- 패널 간 간격 최소화 (시선 이동 줄이기)

### 반응형
- 모바일에서 가로 바 → 세로 바 변환 검토
- 범례 → 직접 라벨로 전환 (모바일에서 범례 공간 부족)

## 🎯 우리 대시보드 적용 계획

| 현재 | 문제 | 개선 |
|---|---|---|
| 멀티라인 독립정규화 | 거짓 상관 | 스몰 멀티플 3~4 패널 |
| 모든 지표 한 차트 | 스파게티 | 그룹별 분리 (트래픽/광고/전환/매출) |
| 본체 3대를 라인으로 | 무의미 | 숫자 카드 + 도트 |
| 색상 랜덤 | 의미 없음 | 고정 의미 매핑 |
| 제목 없음 | 뭘 봐야 하는지 모름 | "답" 제목 자동 생성 |

### 차트 그룹 (4패널 스몰 멀티플)
1. **🔍 트래픽** — 세션/유저/신규유저 (라인, 같은 단위)
2. **💰 광고** — 광고비(바) + CTR/CPC(라인) → 바+라인 콤보
3. **🎯 전환** — 본체(바) + 체험(바) + 전환율(라인) → 바+라인 콤보
4. **📈 매출/효율** — 매출(바) + ROAS(라인) + CPA(라인) → 스몰 멀티플 2개
5. **🔀 인덱스 뷰** — 모든 지표 W기준=100 변화율 비교 (토글)

### 인풋 액션 레이어 (차트 하단)
- 캠페인 간트: 라이브=실선, 중단=점선
- 콘텐츠 마커: 📸(IG) 🎬(YT) 📝(블로그) 업로드 시점
- 이벤트: 수직선 + 라벨
