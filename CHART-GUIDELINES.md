# 차트 시각화 가이드라인

> 출처: Storytelling with Data (Cole Nussbaumer Knaflic), Datawrapper 블로그, Tableau Best Practices, 2011 Dual-Scale Data Charts 연구 (Isenberg et al.)

---

## 🔴 핵심 원칙

### 1. 듀얼 Y축은 쓰지 않는다
- **문제:** 두 축의 스케일이 자의적이라 교차점/상관관계가 거짓 인상을 줌
- **2011 연구 결과:** 15명 중 14명이 "가장 혼란스럽고 해독에 집중력이 많이 든다"고 평가
- **Cole Nussbaumer:** "다른 스케일을 같은 공간에 넣는 순간 관계에 대한 거짓 인상이 생긴다"

### 대안 (우선순위)
1. **스몰 멀티플** — 같은 시간축, 각자 차트. 위아래 쌓기
2. **인덱스 차트** — 시작점=100%으로 통일, 변화율 비교 (절대값은 호버/라벨)
3. **한 지표 집중 + 주석** — 중요한 놈 하나만 차트, 나머지는 텍스트 주석
4. **바+라인 콤보** — 절대값은 바, 비율은 라인 (단, 단위가 명확히 다를 때만)

---

### 2. 차트 유형 선택 기준

| 보고 싶은 것 | 차트 유형 | 예 |
|---|---|---|
| 시간에 따른 추이 | 라인 차트 | 주차별 세션, 광고비 |
| 카테고리 비교 | 가로 바 차트 | VP별 태깅 분포 |
| 구성 비율 | 스택 바 / 도넛 | 채널별 광고비 비중 |
| 상관관계 | 스캐터플롯 | 광고비 vs 전환 |
| 범위/분포 | 히스토그램 / 박스플롯 | 빈도 분포 |
| 일정/구간 | 간트 차트 | 캠페인 라이브 기간 |
| 작은 절대값 (0~10) | 바 차트 or 도트 | 본체 판매 3대 |
| 큰 스케일 차이 비교 | 인덱스 차트 (변화율) | 광고비 vs ROAS |

### 3. 스케일이 다른 지표 비교 방법

**❌ 하면 안 되는 것:**
- 광고비(₩100만)와 본체판매(3대)를 같은 Y축에 넣기
- 독립 스케일 정규화로 "모양만 비슷하게" 보여주기 (거짓 상관)

**✅ 해야 하는 것:**
- **스몰 멀티플:** 동일 시간축, 위아래로 나란히 배치. 각자 적절한 Y축
- **인덱스 차트:** 시작 주차=100, 이후 변화율(%)로 비교
- **KPI 카드 + 추이:** 숫자는 카드로, 추이만 차트로

### 4. 색상 원칙

- **의미 있는 색상:** hit=초록, miss=빨강 등 직관적 매핑
- **같은 계열은 같은 톤:** Meta 관련 = 파란 계열, 전환 = 초록 계열
- **3~5색 이내:** 7개 이상이면 구분 불가
- **대비:** 배경과 충분한 대비, 색약 배려 (빨-초 대신 파-주황)

### 5. 텍스트/주석 원칙

- 차트 제목 = "질문의 답" ("세션은 감소하지만 전환율은 상승")
- 중요 포인트에 **직접 라벨** (범례보다 라벨이 빠름)
- **이벤트 마커:** 캠페인 시작/중단, 스포엑스 등 주요 사건은 수직선+주석
- 축 라벨 최소화, 단위는 한번만

### 6. 레이아웃

- 가장 중요한 차트 = 좌상단
- 한 뷰에 차트 3~4개 이내
- **같은 시간축 = 수직 정렬** (X축 공유, Y축 각자)

## 🎯 우리 대시보드 적용 계획

### 현재 문제
- 모든 지표를 하나의 멀티라인 차트에 독립 정규화로 쑤셔넣음
- 스케일이 다른 지표끼리 "모양만 비슷하게" 보여서 거짓 상관 유발
- 작은 수치(본체 3대)가 선 차트에서 의미 없는 직선

### 개선 방향
1. **메인 추이 차트** — 트래픽 계열만 (세션/유저/신규유저)
2. **광고 효율 차트** — 광고비/노출/클릭/CTR (같은 스케일 가능)
3. **전환 차트** — 본체판매(바) + 체험(바) + CPA(라인) → 바+라인 콤보
4. **매출/ROAS** — 매출(바) + ROAS(%) → 스몰 멀티플 또는 바+라인
5. **인풋 액션 레이어** — 차트 하단 간트(캠페인) + 마커(콘텐츠)
6. **인덱스 뷰** — 모든 지표 W6=100 기준 변화율 비교 (토글)

### Chart.js 전환 이유
- 호버 툴팁, 줌, 범례, 반응형 자동 처리
- 바+라인 콤보, 스택, 어노테이션 플러그인 기본 제공
- 스몰 멀티플 구현 용이
