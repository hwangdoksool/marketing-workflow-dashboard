## 1. 왜 워크플로우인가 (10분)

**목표:** "지금까지 뭐가 문제였고, 왜 이 도구를 만들었는지" 공감대 형성

### 보여줄 숫자 3개
1. **설계서 0건** — 2개월간 40+건 액션 중 "왜 이걸 하는가"를 기록한 적 없음
2. **반복 비용** — 리플렛 4회, 리포트 5회, 블로그 5회 = 방향 합의 없이 시작한 결과
3. **시그널 포착 = 대표님 100%** — 마케터가 데이터에서 문제를 잡은 적 없음

### 핵심 메시지
> "지금까지는 대표님이 발견→지시→검수까지 다 했습니다. 이제 마케터가 데이터를 보고 시그널을 잡고, 가설을 세우고, 결과를 판정하는 구조를 만들겠습니다."

### 참조 자료
- `docs/RETROSPECTIVE-WORKFLOW.md` — 전체 소급 회고
- `docs/2MONTH-REVIEW.md` — 타임라인 원본

## 2. 구매자는 어떻게 사는가 (10분)

**목표:** 우리 고객의 실제 구매 행동을 데이터로 공유

### 보여줄 것
- `reports/buyer-analysis-2026-02-03.html` (정본 보고서)
  
### 핵심 포인트 5개
1. **13건, 전원 모바일, 간편결제** — 결제 UX가 전환에 직결
2. **재방문자 63%가 결제 완료** — 첫 방문에서 안 사고, 돌아와서 산다
3. **Meta가 양, 검색이 질** — 재방문 유입 2,561세션(Meta) vs 체류 13분(네이버 검색)
4. **커뮤니티 재방문 43%** — 후기가 구매 검토의 핵심 콘텐츠
5. **가설 3가지** — 광고인지→검색구매, 이벤트트리거, 즉시구매

### 방법론 교육 (자연스럽게)
- "이 보고서에서 🟢/🟡/🔴 표시가 있습니다. 데이터의 신뢰도를 구분한 겁니다."
- "예를 들어 '재방문자 63% = 구매자'라고 하면 틀립니다. 정확히는 '결제 완료자의 63%가 이전 방문 이력 보유'입니다."
- 자세한 방법론: `shared/methodologies/GA4-IMWEB.md`

## 4. 같이 해보기 (15분)

**목표:** 마케터가 직접 시그널 등록 → 설계서 작성까지 체험

### 진행
1. **W13 리포트를 같이 본다** (대시보드에서)
2. **"이상한 점이 뭐가 보이세요?"** → 마케터가 직접 발견하게 유도
   - 힌트: CPC 추이, 전환율 변화, 채널별 차이 등
3. **시그널 등록** — 마케터가 직접 입력 (또는 에이전트 분석 사용)
4. **설계서 작성** — 가설("~하면 ~할 것이다") + 검증 기준("2주 후 ~가 ~이면 성공")
5. **"이게 이번 주에 돌릴 첫 실험입니다"**

### 예비 시그널 후보 (마케터가 못 찾을 경우 대비)
- Meta CPC 상승 추세 → 소재 피로도?
- 네이버 SA CTR 높은데 클릭수 적음 → 예산 확대?
- 커뮤니티 재방문 43% → 후기 콘텐츠 강화?
