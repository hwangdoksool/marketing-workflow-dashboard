## 📊 전체 요약

### 2개월간 마케팅 액션 총 40+건

| 워크플로우 단계 | 건수 | 비중 | 실제 양상 |
|:---|:---:|:---:|:---|
| Signal (시그널 포착) | 8건 | 20% | **전부 대표님 발견.** 마케터 발견 0건 |
| 설계서 (실험 설계) | 0건 | 0% | 구조 없었음 — 발견 즉시 실행 |
| Make (제작) | 22건 | 55% | 웹 구조 12 + 인프라 10 |
| Live (집행) | 6건 | 15% | 광고 소재 3 + 퍼널 변경 3 |
| Review (검토) | 3건 | 8% | 리포트로만 존재, 실험 판정 없음 |
| Learn (학습 누적) | 1건 | 2% | 스포엑스 회고 1회 |

### 핵심 발견
1. **설계서 0건** — "왜 이걸 하는가"를 기록한 적 없음. 발견→실행 직행
2. **Review-Learn 부재** — 결과가 좋았는지 나빴는지 판정 안 함. 다음에 같은 실험 반복 가능
3. **Signal 독점** — 시그널 포착자 = 대표님 100%. 워크플로우의 첫 단추가 1인에게 집중

## 🔄 워크플로우 단계별 진단

### 1. Signal (시그널 포착)
- **발생 건수:** 8건 이상 (암묵적 시그널 포함하면 더 많음)
- **포착자:** 대표님 100%, 마케터 0%, AI 보조 0%
- **유형 분포:**
  - 데이터 기반: 3건 (UTM 미설정, 전환율 병목, 리포트 파편화)
  - 프로세스 구멍: 2건 (노쇼, 주문 누락)
  - 기회 포착: 2건 (CTA 강화, 데이터 인프라 필요)
  - 현장 관찰: 1건 (스포엑스)
- **🔴 문제:** 마케터가 데이터를 보고 시그널을 잡은 적이 없음. 워크플로우에서 "주간 리포트 발행 → 마케터가 시그널 등록"이 구조적으로 강제되어야 함

### 2. 실험 설계서
- **작성 건수:** 0건
- **🔴 문제:** 가장 큰 구조적 공백. "왜 이걸 하는가", "성공 기준이 뭔가"가 기록된 적 없음
- **비용:** 블로그 5회, 리플렛 4회, 리포트 5회 반복 = 설계서 부재의 직접 비용
- **반사실적 추정:** 설계서가 있었다면 반복 횟수 50%+ 감소 가능

### 3. Make (제작)
- **건수:** 22건 (전체의 55%)
- **유형:** 웹 구조 12건, 인프라 10건, 광고 소재 3건, 콘텐츠 4건, 오프라인 3건
- **🔴 문제:** "광고 소재 = 마케팅"이라는 프레임에서는 93%의 액션이 보이지 않음
- **교훈:** 워크플로우의 "액션 유형"에 웹/퍼널/인프라/콘텐츠/오프라인 전부 포함 필수 (→ 반영 완료)

### 4. Live (집행)
- **건수:** 6건
- **광고:** Meta ASC 캠페인 연속 집행 (W9부터), 네이버 검색광고 병행
- **웹:** 홈페이지 변경사항은 배포 = Live
- **퍼널:** 체험 시스템 2차례 대전환
- **🔴 문제:** Live 시점에 "이 실험의 검증 기간은 X주"가 설정된 적 없음

### 5. Review (검토)
- **건수:** 3건 (리포트 형태로만)
- **🔴 문제:** "결과가 좋았는가 / 가설대로였는가" 판정이 거의 없음
- **예외:** UTM 수정(정상화 확인), 주문 알림(작동 확인) — 인프라는 binary 판정이라 자연스러웠음
- **마케팅 실험에 대한 Review는 0건** — 비교표 넣고 전환율이 올랐는지 안 올랐는지 정식으로 판정한 적 없음

### 6. Learn (학습 누적)
- **건수:** 실질적으로 0건 (기록된 학습)
- **🔴 문제:** 2개월간 쌓인 경험이 개인 기억에만 존재. 형은님 계약 종료 시 증발
- **소급 학습 후보:**
  - "CTA 중앙팝업 + 다크 배경 = 세션당 노출 1회 최적"
  - "비교표 추가 후 /buy 체류시간 증가 (표본 부족이지만 방향성)"
  - "UTM 미설정 → 채널 분석 불가 (프로세스 체크리스트 필요)"
  - "체험 결제 선행 → 노쇼율 0% (2주 관찰)"
  - "리플렛/리포트 반복 = 방향 합의 없이 시작한 결과"
  - "스포엑스 묠니르 120kg = 구경꾼 어트랙터이지 구매 트리거 아님"

## 🎯 워크플로우로 전환 시 기대 효과

### Before (과거 2개월)
```
대표님 머릿속 → 즉시 실행 → (결과 모름) → 다음 실행
```

### After (워크플로우 적용 시)
```
리포트 발행 → 마케터+대표님 시그널 등록 → 설계서(가설+검증기준) → 제작 → Live → Review(2축 판정) → Learn(누적)
```

### 소급 시뮬레이션: "비교표 추가" 액션

**실제 (설계서 없이):**
1. 대표님: "전환율 낮다, 비교표 넣자" → 바로 제작 → 배포 → 효과 불명

**워크플로우 적용 시:**
1. **Signal:** W9 리포트에서 상품조회→결제시작 전환율 1.6% 감지 (AI 자동 플래그)
2. **설계서:** "가설: 경쟁 비교 정보 부족 → 비교표 추가 시 결제시작 전환율 2배 예상. 검증 기준: 2주 후 전환율 3%+ 도달"
3. **Make:** 비교표 디자인+구현
4. **Live:** 3/1 배포
5. **Review:** 2주 후(3/15) 판정 — 결제시작 전환율 1.6%→2.4% (목표 미달이나 상승)
6. **Learn:** "비교표 = 전환율 50% 개선. 추가 개선 여지: 가격 표기 방식, 할부 안내"

→ **다음 번에 유사한 시그널이 왔을 때 "비교표는 효과 있었으니 이번엔 할부 안내를 추가해보자"로 이어짐**

### L001: CTA 중앙팝업
- **기간:** W8(2/17) Live
- **가설:** (암묵적) 메인 CTA 노출이 약하다
- **결과:** 세션당 1회 노출 + 다크 배경 적용
- **학습:** 팝업 과다 → 이탈. 1회가 최적. 패턴 opacity 0이 가독성 최고

### L002: UTM 미설정 발견
- **기간:** W9(2/24)
- **가설:** 채널별 성과 추적 불가 상태
- **결과:** 수정 후 Display/Paid Social 분리 가능
- **학습:** 광고 집행 시작 시 UTM 체크리스트 필수. 마케터 자체 검증 프로세스 필요

### L003: 상세페이지 비교표+결제허들
- **기간:** W10(3/1) Live
- **가설:** 전환율 낮은 이유 = 경쟁 비교 + 결제 심리 장벽
- **결과:** 상품조회→결제시작 전환율 미세 상승 (표본 부족)
- **학습:** 방향은 맞으나 효과 크기 판정 불가. 장기 관찰 필요

### L004: 체험 결제 선행 전환
- **기간:** W11(3/11) Live
- **가설:** 무료 신청 → 노쇼 → 결제 선행하면 노쇼 감소
- **결과:** 결제 선행 후 노쇼율 대폭 감소
- **학습:** 금전적 커밋 = 노쇼 방지에 효과적. 단, 신청 전환율은 하락 가능 (트레이드오프)

### L005: 리플렛/리포트 반복 비용
