# 마케팅 워크플로우 전략 설계 (초안)

> **목적**: 가설→실험→검증→학습 루프를 구조적으로 강제하는 워크플로우
> **작성**: 2026-04-02 (고릴 🦍)
> **상태**: 대표님 검토용 초안

---

## 전제 (대표님 지시 4가지)

1. **자동화된 상세 리포트** → 데이터 기반 인사이트 도출
2. **전 과정 명문화/데이터화** → 특히 의도 설계 + 결과 분석
3. **연속성 + 레거시** → 단편적 종료 없이 누적
4. **Creative Brief로 소통 효율화** → Brief 단계에서 논의 종결

---

## 1. 워크플로우 전체 흐름

```
[SIGNAL] → [BRIEF] → [MAKE] → [LIVE] → [REVIEW] → [LEARN]
  ↑                                                    │
  └────────────────────────────────────────────────────┘
```

### 각 단계 정의

| # | 단계 | 누가 | 뭘 하나 | 산출물 |
|---|------|------|---------|--------|
| 0 | **Signal** | AI+리더 | 데이터에서 기회/위협 신호 포착 | 시그널 카드 |
| 1 | **Brief** | 마케터+리더 | 가설 설계 + 방향 협의 (여기서 끝) | Creative Brief |
| 2 | **Make** | 마케터 | 소재 제작 (Brief 범위 내 자율) | 소재 |
| 3 | **Live** | 마케터 | 게시/집행 + 라이브 메타데이터 기록 | 라이브 기록 |
| 4 | **Review** | AI+리더 | 자동 리포트 + 가설 검증 | 주간 리뷰 |
| 5 | **Learn** | 전원 | 인사이트 확정 + 다음 Signal로 연결 | 학습 카드 |

---

## 2. 각 단계 상세

### Stage 0: Signal (시그널 — 새로 추가)

**왜 필요한가:**
현재 구조에서 빠진 것 = "Brief를 왜 쓰게 됐는가"의 출발점.
Brief가 허공에서 나오면 "또 새로운 일" 이 되고,
데이터 신호에서 출발하면 "이것 때문에 해야 하는 일"이 된다.

**시그널 유형:**
- 🔴 위협: 소재 피로도 상승 (CTR↓ CPC↑), 퍼널 병목 악화, 이탈률 급등
- 🟢 기회: 특정 타겟/VP 조합 초기 반응 좋음, 신규 채널 성과, 오가닉 신호
- 🔵 외부: 시즌/이벤트, 경쟁사 움직임, 업계 트렌드
- 🟡 학습: 이전 Brief의 검증 결과에서 파생된 후속 실험 아이디어

**자동화 범위:**
- AI가 매일/매주 데이터 스캔 → 시그널 자동 감지
- 예: "ASC 캠페인 CTR이 3일 연속 하락 중 (6.2→5.4→4.8)" → 시그널 카드 자동 생성
- 마케터/리더도 수동으로 시그널 등록 가능 (직관, 외부 정보)

**핵심:**
시그널이 있어야 Brief가 "왜 지금 이걸 해야 하는가"에 답할 수 있다.
Brief의 `배경` 필드가 시그널 카드를 참조하는 구조.

---

### Stage 1: Brief (크리에이티브 브리프)

**현재 양식 + 보강안:**

| 필드 | 설명 | 필수 | 보강 |
|------|------|------|------|
| **배경/시그널** | 왜 지금 이걸 하는가 | ✅ 신규 | 시그널 카드 링크 |
| 페르소나 | 누구에게 | ✅ | 기존 유지 |
| JTBD | 어떤 문제를 | ✅ | 기존 유지 |
| VP | 어떤 가치를 | ✅ | 기존 유지 |
| 소재 구성 | 어떤 형태로 | ✅ | 기존 유지 |
| 채널 | 어디에 | ✅ | 기존 유지 |
| **가설** | 기대 효과 | ✅ | 기존 유지 |
| **검증 기준** | 언제, 무슨 지표로 | ✅ | 기존 유지 |
| **참조 Brief** | 이전 유사 실험 | 선택 | 신규 — 레거시 연결 |
| **리더 코멘트** | 방향 협의 결과 | 선택 | 신규 — 협의 기록 |

**프로세스 규칙 (진단 보고서 반영):**
- Brief 단계: 자유롭게 수정 — **모든 전략 논의는 여기서 종결**
- 제작 중: Brief 범위 내 마케터 자율, 리더 개입 금지
- 완성 후: Brief 대비 정합성만 확인
- → "의도가 뭐냐" 질문이 구조적으로 사라짐

---

### Stage 2: Make (제작)

마케터 자율 영역. 시스템이 강제하는 건 없음.
Brief 범위 내에서 자유롭게 제작.

**기록되는 것:**
- Brief 상태: Draft → In Progress (선택사항, 추적용)
- 제작 기간 자동 계산 (Brief 생성일 ~ Live 전환일)

---

### Stage 3: Live (라이브)

**마케터가 하는 것:**
- Brief 상태를 Live로 변경
- 라이브 메타데이터 입력: 라이브 날짜, 캠페인/소재 ID (Meta/네이버 등)

**시스템이 하는 것:**
- 라이브 날짜부터 검증 기간 자동 카운트
- 해당 캠페인/소재의 성과 데이터 자동 수집 시작
- 검증 기간 도래 시 알림

---

### Stage 4: Review (주간 리뷰)

**자동 생성 리포트 구조 (노션 운영 가이드 4부 그대로):**

| 파트 | 내용 | 자동화 수준 |
|------|------|------------|
| 0부: 구매자 역추적 | 구매 건별 유입경로, 행동 타임라인 | 🟡 반자동 (GA4 데이터 + 수동 매칭) |
| 1부: 채널별 성과 | Meta/GA4/네이버/인스타/아임웹 | 🟢 완전 자동 |
| 2부: 가설 검증 | Live Brief의 가설 vs 실제 결과 매칭 | 🟢 자동 매칭 + 수동 판정 |
| 3부: 이번 주 계획 | 시그널 + 학습 기반 다음 Brief 방향 | 🟡 AI 제안 + 수동 확정 |
| 4부: 이슈/블로커 | 리소스, 기술, 의사결정 필요사항 | 🔴 수동 |

**2부(가설 검증)가 핵심 — Brief와 리뷰의 연결고리:**
- Brief의 검증 기준 기간이 도래한 Brief 자동 리스트업
- 해당 기간의 실제 데이터 자동 풀링
- 마케터/리더가 판정: ✅ 유효 / ❌ 기각 / ⏳ 미확정
- 판정 근거 기록 필수

---

### Stage 5: Learn (학습 — 레거시 축적)

**이게 기존 대시보드에 없던 핵심 단계.**

학습 카드:
- 출처 Brief 링크
- 판정 결과 + 근거
- **"다음에 시도할 것"** — 후속 실험 아이디어 → 다시 Signal로
- **태그**: VP별, 페르소나별, 채널별, 소재구성별

**축적 효과:**
- "VP2(안전) + PS1(바쁜직장인)은 3번 시도, 2번 유효" 같은 패턴 발견
- "비교 프레임 소재는 CTR은 높지만 전환은 낮다" 같은 구조적 학습
- 마케터가 바뀌어도 학습이 남음
- 시간이 지날수록 Brief 작성 시 "이전에 비슷한 시도에서 뭐가 나왔는지" 자동 추천

---

## 3. 레거시 축적 구조 (요구사항 #3)

```
Signal ←── Learn에서 파생
  │
Brief ←── 참조 Brief (과거 유사 실험 링크)
  │
Live ──→ 데이터 자동 수집
  │
Review ──→ Brief별 결과 매칭
  │
Learn ──→ 태그별 패턴 축적
  │
  └──→ 다음 Signal / 다음 Brief의 참조 자료
```

**끊기지 않는 고리:**
- 모든 Brief는 이전 Brief를 참조할 수 있음
- 모든 학습 카드는 다음 시그널의 근거가 됨
- VP/페르소나/채널 축별로 성과가 누적 → 시간이 지날수록 판단 근거가 두꺼워짐

## 5. 소통 효율 설계 (요구사항 #4)

**진단 보고서의 핵심 발견 반영:**

| 기존 문제 | 워크플로우에서의 해결 |
|----------|---------------------|
| "의도가 뭐냐"에서 시작하는 피드백 | Brief에 의도가 명문화되어 있음 |
| 리더의 반복 질문 | Brief에서 합의 → "Brief 참조"로 종결 |
| 마케터의 논점 이탈 | Brief 필수 필드가 핵심을 강제 |
| 완성물에서 기획 논의 재개 | 제작 중 개입 금지 규칙 |
| 성공/실패 재현 불가 | 학습 카드에 근거 축적 |
| "이미 설명했다" vs 실제 이해 | Brief가 유일한 진실 소스 |

**피드백 프로토콜 (진단 보고서 5-2 그대로):**
- Brief 단계: 자유 수정 → 여기서 전략 논의 종결
- 제작 중: 마케터 자율
- 완성 후: Brief 대비 정합성만
- 라이브 후: 가설 검증, 학습

## 6. 효과/효율 극대화 전략 (추가 제안)

### 6-1. "시그널 → Brief" 자동 연결

마케터가 매주 "뭘 해야 하지"로 시작하는 대신,
시스템이 "이런 신호가 있다 → Brief 작성하시겠습니까?"로 시작.

예:
> 🔴 ASC 캠페인 CTR 3일 연속 하락 (6.2→5.4→4.8)
> → 소재 교체 Brief를 작성하시겠습니까? [Brief 작성]

마케터의 의사결정 부담 감소 + 데이터 기반 행동 자연스럽게 유도.

### 6-2. Brief 템플릿 프리셋

매번 빈칸에서 시작하면 허들이 높다.
자주 쓰는 조합을 템플릿으로:

- "소재 교체" 템플릿 — 피로도 신호 대응
- "신규 타겟 테스트" 템플릿 — 새 페르소나 실험
- "퍼널 최적화" 템플릿 — 전환율 개선 실험
- "채널 확장" 템플릿 — 새 채널 테스트

### 6-3. 주간 리뷰 = 30분 미팅 대체

리포트가 자동 생성되면 미팅 시간이 절반으로 줄어든다.
- 데이터 공유: 0분 (대시보드에 이미 있음)
- 가설 검증 판정: 10분
- 다음 주 Brief 방향: 15분
- 이슈: 5분
- 합계: 30분 (현재 2시간+ → 30분)

### 6-4. 학습 그래프 (장기)

Brief 축적 → VP×페르소나×채널 3차원 매트릭스 자동 생성.
"어떤 조합을 아직 안 해봤는가" 빈칸이 보이면,
그 자체가 다음 실험의 시그널이 된다.

## 7. 구현 우선순위

| 순서 | 할 일 | 이유 |
|------|-------|------|
| **1** | Brief 양식 확정 + 상태 흐름 재정의 | 전체 구조의 중심축 |
| **2** | 주간 리포트 자동 생성 | 리뷰 부담 제거 → 채택률 결정 |
| **3** | Signal 자동 감지 | Brief의 출발점 |
| **4** | Brief↔성과 자동 매칭 | 검증 자동화 |
| **5** | 학습 카드 + 패턴 분석 | 레거시 축적 |
| **6** | 대시보드 UI 반영 | 위 1~5가 확정된 후 |
