# KPI 계산법 정리

> 마케팅 워크플로우 대시보드 주간 리포트에 사용하는 지표와 계산식
> 데이터 신뢰도: 🟢 Tier 1 (확정), 🟡 Tier 2 (높은 신뢰 추론), 🔴 Tier 3 (가설)

---

## 💰 ROAS (Return On Ad Spend) — 광고 수익률

```
ROAS = 아임웹 실매출 / 광고비 × 100%
```

| 항목 | 출처 | 신뢰도 |
|:---|:---|:---:|
| 매출 | 아임웹 결제 완료 금액 (`orders.json`) | 🟢 |
| 광고비 | Meta Ads API + 네이버 검색광고 API 합산 | 🟢 |

- **의미:** 광고비 100만원 쓰면 얼마 벌었나 (높을수록 좋음)
- **예시:** 매출 ₩10,290,000 ÷ 광고비 ₩1,058,507 = **ROAS 972%** → 광고비 1원당 9.7원 벌었다
- **주의:** GA4 구매건수 × 평균 객단가로 추정하지 않음. 아임웹 실매출 사용.
- **한계:** 해당 주 결제 완료 기준이라, 광고 본 시점과 구매 시점에 시차가 있음 (🟡)

---

## 🎯 CPA (Cost Per Acquisition) — 고객 획득 비용

```
CPA = 총 광고비 / GA4 구매 건수
```

| 항목 | 출처 | 신뢰도 |
|:---|:---|:---:|
| 광고비 | Meta + 네이버 합산 | 🟢 |
| 구매 건수 | GA4 purchase 이벤트 | 🟡 |

- **의미:** 고객 1명 데려오는 데 얼마 썼나 (낮을수록 좋음)
- **예시:** ₩1,058,507 ÷ 6건 = **CPA ₩176,418**
- **주의:** GA4 purchase ≠ 아임웹 주문 건수 (GA4는 이벤트 발화 기준, 중복/누락 가능)
- **CPA vs ROAS:** 동전의 양면이지만 객단가에 따라 의미가 달라짐
  - 객단가 높으면 (룸핏 ₩348만) CPA가 높아도 ROAS는 충분할 수 있음
  - 객단가 낮으면 CPA가 낮아도 ROAS가 적자일 수 있음

## 📊 기본 지표

### 세션 / 유저 / 신규 유저
- **출처:** GA4 (🟢)
- 세션 = 방문 횟수 (같은 유저가 2번 오면 2세션)
- 유저 = 고유 방문자 (쿠키 기준, 기기 바뀌면 다른 유저로 집계)
- 신규 유저 = 해당 기간 내 첫 방문

### 전환율 (Conversion Rate)
```
전환율 = GA4 구매 건수 / 세션 × 100%
```
- **주의:** "세션 전환율"이지 "유저 전환율"이 아님

### CPC (Cost Per Click)
```
CPC = 총 광고비 / 총 클릭수
```
- Meta + 네이버 합산
- 의미: 클릭 1회당 비용

### CTR (Click-Through Rate)
```
CTR = 총 클릭수 / 총 노출수 × 100%
```
- 의미: 광고 본 사람 중 몇 %가 클릭했나

## 📈 퍼널 지표 (GA4 기준)

```
페이지뷰 → 상품조회(view_item) → 장바구니(add_to_cart) → 결제정보(add_payment_info) → 결제시작(begin_checkout) → 구매(purchase)
```

- 각 단계 간 전환율 = 다음 단계 / 현재 단계 × 100%
- **주의:** GA4 이벤트 발화 순서가 실제 사용자 행동과 다를 수 있음 (🟡)

## ⚠️ 계산 시 주의사항

1. **GA4 구매 ≠ 아임웹 주문** — 반드시 교차 확인
2. **쿠키 기반 유저** — 디바이스/브라우저 바뀌면 다른 유저로 집계
3. **시차 문제** — 광고 본 날 ≠ 구매한 날 (ROAS에 영향)
4. **Meta display vs Paid Social** — 3/11 이후 UTM 변경으로 Meta 트래픽 = Display + Paid Social 합산
5. **1,000원 이하 주문 제외** — 테스트 주문 필터링
