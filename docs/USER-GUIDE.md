# 마케팅 워크플로우 대시보드 — 사용 가이드

> **마케터를 위한 운영 매뉴얼**
> 버전: v1.0 | 2026-04-02

---

## 🎯 이 대시보드는 뭔가요?

**"가설 → 실험 → 검증 → 학습"** 사이클을 돌리기 위한 도구입니다.

핵심은 **Creative Brief**입니다.
Brief 없이는 소재를 만들 이유가 없고, Brief가 있으면 결과를 검증할 수 있습니다.

---

## 📌 화면 구성 (4개 탭)

| 탭 | 이름 | 용도 |
|----|------|------|
| 📊 | 주간 리뷰 | 이번 주 성과 한눈에 보기 (KPI, 퍼널, 구매자, 인사이트) |
| 🔍 | 구매자 역추적 | 구매 건별 유입경로 → 전환 타임라인 (준비 중) |
| 📋 | **Brief 관리** | ⭐ 핵심 탭. Brief 작성/관리/검증 |
| 📈 | 종합 분석 | 8주 회고 데이터, 장기 트렌드 |

> 💡 오른쪽 상단 **라이트 모드/다크 모드** 버튼으로 테마 전환 가능

## 📋 Brief 관리 — 핵심 사용법

### Brief란?

소재(광고/콘텐츠)를 만들기 전에 작성하는 **1장짜리 기획서**입니다.
"왜 이걸 만드는가", "뭘 검증하려는가"를 명문화합니다.

### 새 Brief 작성하기

1. **📋 Brief 관리** 탭 클릭
2. 오른쪽 상단 **`+ 새 Brief`** 버튼 클릭
3. 양식 작성 (모든 항목 필수!)

| 필드 | 설명 | 예시 |
|------|------|------|
| **제목** | 캠페인/소재 이름 | "4월 1주차 릴스 — 홈트 시연" |
| **페르소나** | 누구를 타겟하는가 | PS1 바쁜직장인 |
| **JTBD** | 이 사람이 해결하려는 일 | JT5 시간효율 |
| **VP** | 우리가 전달할 가치 | VP1 자유해방 |
| **소재 구성** | 영상/이미지 구성 방식 | 문제→솔루션 |
| **채널** | 어디에 올릴 것인가 | Meta전환, 인스타릴스 등 |
| **가설** | 이 조합으로 어떤 결과를 기대하는가 | "PS1+VP1 조합이 CTR 5% 이상 달성할 것" |
| **검증 기준** | 어떤 숫자로, 언제 판단할 것인가 | "7일 후 CTR 5% 이상이면 유효" |

4. **저장** 클릭 → Draft 상태로 등록

> ⚠️ **가설과 검증 기준은 반드시 구체적으로!**
> - ❌ "효과 있을 것이다" → 너무 모호
> - ✅ "CTR 5% 이상, 7일 후 판단" → 이래야 검증 가능

---

### Brief 상태 흐름

```
Draft (회색) → Live (파랑) → Verified (초록/빨강/노랑)
```

| 상태 | 색상 | 의미 | 언제 바꾸나 |
|------|------|------|------------|
| **Draft** | 🔘 회색 | 기획 중 / 협의 전 | Brief 처음 작성 시 |
| **Live** | 🔵 파랑 | 소재 제작 완료, 라이브 중 | 소재가 실제로 게시/집행되면 |
| **✅ 유효** | 🟢 초록 | 가설이 맞았다 | 검증 기준 충족 시 |
| **❌ 기각** | 🔴 빨강 | 가설이 틀렸다 | 검증 기준 미달 시 |
| **⏳ 미확정** | 🟡 노랑 | 아직 판단 불가 | 데이터 부족 or 기간 미도래 |

### 상태 변경하기

1. Brief 카드 클릭 → 상세 모달 열림
2. 상단의 **드롭다운**에서 새 상태 선택 → 즉시 반영
3. Verified 상태(유효/기각/미확정)로 바꾸면 **검증 결과 기록** 입력란이 나타남
4. 결과를 적고 **결과 저장** 클릭

> 💡 검증 결과에는 "왜 유효/기각인지"를 꼭 남겨주세요.
> 이게 쌓여야 다음 Brief 작성 시 참고할 수 있습니다.

---

### Brief 수정 / 삭제

- **수정**: 상세 모달 하단 `수정` 버튼 → 양식 다시 열림
- **삭제**: 상세 모달 하단 `삭제` 버튼 → 확인 후 삭제 (복구 불가!)

---

### 검색 & 필터

| 기능 | 사용법 |
|------|--------|
| **검색** | 검색창에 키워드 입력 → 제목, 페르소나, JTBD, 가설에서 검색 |
| **필터** | 전체 / Draft / Live / Verified 버튼 클릭 → 해당 상태만 표시 |

---

### JSON 내보내기 / 가져오기

Brief 데이터는 브라우저 로컬 저장소에 저장됩니다.
**다른 기기로 옮기거나 백업할 때** 사용합니다.

| 버튼 | 기능 |
|------|------|
| **JSON 내보내기** | 현재 모든 Brief를 `.json` 파일로 다운로드 |
| **JSON 가져오기** | `.json` 파일을 선택하여 Brief 일괄 불러오기 |

> ⚠️ 가져오기 시 **기존 데이터가 덮어쓰기** 됩니다!
> 중요한 데이터는 먼저 내보내기로 백업하세요.

## 📊 주간 리뷰 탭

매주 자동 생성되는 성과 리포트입니다.

### 구성

1. **주차 선택** — 상단 버튼으로 주차 이동
2. **요약** — 한 줄 핵심 (보라색 왼쪽 선)
3. **KPI 카드** — 주요 지표 (전주 대비 ▲▼ 표시)
   - 🟢 초록 화살표 = 개선
   - 🔴 빨간 화살표 = 악화
   - CPC, CPA 등 비용 지표는 **반대** (낮을수록 좋음)
4. **섹션별 상세** — 채널 성과, 퍼널, 구매자, 인사이트 등

> 💡 데이터는 `reports.json`에서 불러옵니다. 직접 수정할 필요 없습니다.

## 📈 종합 분석 탭

W6~W13 (8주) 마케팅 종합 회고 데이터입니다.

## ❓ 자주 묻는 질문

### Q. Brief 데이터가 날아갔어요!
A. 브라우저 로컬 저장소에 저장되므로, 브라우저 데이터를 지우면 사라집니다.
→ 정기적으로 **JSON 내보내기**로 백업하세요.

### Q. 다른 컴퓨터에서도 보고 싶어요
A. JSON 내보내기 → 파일 전달 → 다른 컴퓨터에서 JSON 가져오기

### Q. Brief 양식의 드롭다운 항목을 바꾸고 싶어요
A. 황성우 대표님 또는 AI팀에 요청해주세요. (코드 수정 필요)

### Q. 주간 리뷰 데이터는 어떻게 업데이트되나요?
A. AI가 매주 자동으로 `reports.json`을 갱신합니다. 마케터가 직접 할 일은 없습니다.

## 🔄 Brief 작성 체크리스트

새 소재/캠페인 기획 시 이 순서대로:

- [ ] 📋 Brief 작성 (모든 필수 필드)
- [ ] 🤝 리더와 Brief 협의 (방향 확정)
- [ ] 🎨 소재 제작
- [ ] 🔵 상태를 **Live**로 변경
- [ ] ⏰ 검증 기준 기간 도래 시 데이터 확인
- [ ] 🟢🔴🟡 상태를 **Verified**로 변경 + 검증 결과 기록
- [ ] 📝 다음 Brief에 학습 반영
