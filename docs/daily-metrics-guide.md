# 일별 지표 해석 가이드

> **마케팅 워크플로우 대시보드 > 일별 지표 탭**의 각 지표 정의, 데이터 출처, 해석 방법 및 주의사항을 상세히 설명합니다.

---

## 📊 지표 개요

일별 지표는 **사이트 트래픽(GA4)**와 **실제 판매(아임웹 정본)**를 통합하여 보여주는 일간 성과 대시보드입니다.

| 구분 | 데이터 소스 | 설명 |
|------|------------|------|
| **트래픽 지표** | GA4 (Google Analytics 4) | 사이트 방문 및 사용자 행동 |
| **판매 지표** | 아임웹 정본 (`imweb_body_sales.json`) | 실제 주문 및 매출 데이터 |
| **채널 세부** | GA4 + Meta/Naver API | 채널별 트래픽 및 전환 분석 |

## 📈 지표 상세 정의

### 1. 트래픽 지표 (GA4 기준)

#### Sessions (세션)
- **정의**: 사용자가 사이트를 방문하여 30분간 활동이 없을 때까지의 방문 단위
- **측정 기준**: GA4 전체 사이트 (roomfit.kr 및 모든 하위 페이지)
- **활용**: 마케팅 활동의 트래픽 발생 효과 측정
- **주의**: 동일 사용자의 여러 방문 = 복수 세션으로 카운트

#### Users (사용자)
- **정의**: 고유한 사용자 수 (기기/브라우저 기반 중복 제거)
- **측정 기준**: GA4 first_visit 이벤트 기반
- **활용**: 실제 도달(reach) 규모 파악
- **주의**: 크로스 디바이스(모바일→PC)는 별도 사용자로 인식될 수 있음

#### New Users (신규 사용자)
- **정의**: 처음 방문한 사용자 수
- **활용**: 신규 유입 채널 효과, 브랜드 인지도 확산 측정
- **계산**: Users 중 첫 방문자 비율 = 신규 사용자 비율

#### Pageviews (페이지뷰)
- **정의**: 페이지 조회 총 횟수
- **활용**: 콘텐츠 소비량, 사용자 관심도 측정
- **주의**: 한 세션에서 여러 페이지 볼 수 있음 (세션당 평균 페이지뷰로 활용 권장)

#### Bounce Rate (이탈률)
- **정의**: 단일 페이지 방문 후 이탈한 세션 비율 (GA4 기준: engagement 없는 세션)
- **정상 범위**: 30-60% (업종/채널에 따라 상이)
- **활용**: 랜딩 페이지 품질, UX 문제 진단
- **해석**: 
  - 높음(>70%): 페이지 품질 문제 또는 잘못된 타겟팅
  - 낮음(<30%): 우수한 참여도 (단, 목적에 맞는지 확인 필요)

#### Avg. Session Duration (평균 세션 시간)
- **정의**: 세션당 평균 체류 시간 (초)
- **활용**: 콘텐츠 몰입도, 사이트 유용성 측정
- **주의**: 마지막 이벤트 이후 시간은 측정 불가 (실제보다 짧게 측정됨)
- **정상 범위**: 60-180초 (제품 페이지 기준)

### 2. 판매 지표 (아임웹 정본 기준)

> ⚠️ **절대 규칙**: 판매/매출 데이터는 **아임웹 정본만** 사용합니다.
> - Meta 픽셀 Purchase, GA4 전환 이벤트는 **참고치일 뿐** 전환/CPA 계산 근거로 사용 금지
> - 2/25 이전 픽셀 테스트 주문은 전체 배제

#### Orders (주문 건수)
- **정의**: 아임웹에서 생성된 결제 완료 주문 수
- **데이터 출처**: `imweb_body_sales.json` (아임웹 브라우저 크롤링 기반)
- **포함**: 본체 판매 + 액세서리 판매
- **제외**: 취소/환불 완료 주문, 체험 예약금

#### Body Units (본체 판매 대수)
- **정의**: 룸핏 본체 판매 수량
- **활용**: 핵심 제품 매출 기여도 측정
- **주의**: 복수 구매(2대 이상)는 대수로 집계됨

#### Body Revenue (본체 매출)
- **정의**: 본체 판매 매출 (부가세 포함)
- **계산**: `본체 판매 대수 × 단가(3,480,000원)`
- **주의**: 할인/쿠폰 적용 시 차감된 실결제 금액 기준

#### Accessory Revenue (액세서리 매출)
- **정의**: 매트, 타워 등 액세서리 판매 매출
- **활용**: 제품별 매출 기여도 분석, 크로스셀 효과

#### Revenue (총 매출)
- **정의**: Body Revenue + Accessory Revenue
- **주의**: 체험 예약금(99,000원)은 제외됨

#### Feel Bookings (체험 예약)
- **정의**: 체험 신청 접수 수 (결제 상태 무관)
- **데이터 출처**: Supabase `feel_bookings` 테이블
- **포함**: pending_payment, paid_pending, confirmed 등 전체 상태
- **주의**: 체험 예약금 매출은 별도 집계되지 않음

### 3. 캠페인 지표 (Meta/Naver)

> ⚠️ **중요**: GA4 채널 분류와 실제 광고 소재의 UTM이 다를 수 있음
> - Meta 광고: GA4에서 Display 또는 Paid Social로 분류
> - 3/11 이후 신규 소재는 Paid Social, 기존 소재는 Display로 분류됨
> - 정확한 분석을 위해 Meta Ads API로 실제 UTM 세팅 직접 조회 필요

#### Campaigns (캠페인 수)
- **정의**: Meta Ads + Naver 검색광고에서 활성화된 캠페인 수
- **활용**: 운영 중인 광고 규모 파악

#### Adsets (광고 그룹 수)
- **정의**: 캠페인 내 광고 그룹(세트) 수
- **활용**: 타겟팅 세분화 수준

#### Ads (광고 소재 수)
- **정의**: 실제 노출 중인 광고 소재 수
- **활용**: 크리에이티브 다양성 및 A/B 테스트 규모

#### Spend (광고비)
- **정의**: Meta + Naver 소진 광고비 (KRW)
- **데이터 출처**: Meta Marketing API + Naver 검색광고 API
- **활용**: ROAS, CPA 계산 기반

#### Impressions (노출수)
- **정의**: 광고가 화면에 표시된 총 횟수
- **활용**: 도달 규모, 브랜드 인지도

#### Clicks (클릭수)
- **정의**: 광고 클릭 총 횟수
- **활용**: 관심도, CTR 계산

#### CTR (클릭률)
- **정의**: Clicks ÷ Impressions × 100
- **정상 범위**: 
  - Meta: 0.5-2.0%
  - Naver 검색: 3-15%
- **해석**: 
  - 높은 CTR: 크리에이티브/키워드 효과 우수
  - 낮은 CTR: 크리에이티브 개선 또는 타겟팅 조정 필요

#### CPM (천회 노출당 비용)
- **정의**: (Spend ÷ Impressions) × 1000
- **활용**: 매체 효율성 비교
- **정상 범위**: 
  - Meta: 3,000-10,000원
  - Naver: 1,000-5,000원

#### CPC (클릭당 비용)
- **정의**: Spend ÷ Clicks
- **활용**: 트래픽 확보 비용 효율
- **정상 범위**:
  - Meta: 500-2,000원
  - Naver: 300-1,500원

---

### 4. 전환/효율 지표

#### ROAS (광고비 대비 매출)
- **정의**: Revenue ÷ Spend × 100
- **해석 기준**:
  - 300%+ : 우수
  - 200-300% : 양호
  - 100-200% : 주의
  - 100% 이하 : 개선 필요
- **주의**: 브랜딩 캠페인은 ROAS가 낮을 수 있음 (목적 고려)

#### CPA (고객 획득 비용)
- **정의**: Spend ÷ Orders
- **활용**: 주문 1건당 광고비용
- **해석**: 제품 마진 대비 적정 CPA 산정
- **정상 범위**: 제품 단가의 20-40% (카테고리에 따라 상이)

#### Conversion Rate (전환율)
- **정의**: Orders ÷ Sessions × 100 (또는 Users 기준)
- **정상 범위**: 0.5-3%
- **해석**: 
  - 트래픽 대비 판매 효율
  - 낮을 경우: 트래픽 품질 문제 또는 사이트 개선 필요

## 🔄 WoW (주간 비교) 해석

### 계산 방법
- **기준**: 해당 일자 vs 7일 전 동일 요일
- **공식**: ((오늘 - 7일 전) ÷ 7일 전) × 100

### 색상 기준
| 색상 | 의미 | 판단 |
|------|------|------|
| 🟢 초록 | +10% 이상 | 긍정적 성장 |
| 🟡 노랑 | -10% ~ +10% | 안정적 유지 |
| 🔴 빨강 | -10% 이하 | 주의 필요 |

### 해석 시 주의사항
- **요일 효과**: 주중/주말, 월요일/금요일 패턴 고려
- **이벤트 영향**: 세일, 광고 집행, 휴일 등 외부 요인 체크
- **표본 크기**: 수치가 작을 때(예: 주문 1→2건) 퍼센트 변동 의미 없음
- **계절성**: 월초/월말, 특정 월 패턴 고려

## ⚠️ 데이터 한계 및 주의사항

### 1. 시간 차이
- **GA4 데이터**: 실시간 ~ 24시간 지연
- **아임웹 데이터**: 주문 발생 후 다음날 새벽 업데이트 (06:00 기준)
- **광고 데이터**: Meta/Naver API 1-2시간 지연
- **결과**: 오늘 데이터는 부정확할 수 있음 (어제까지가 확정 데이터)

### 2. 중복/누락 가능성
- **GA4**: 애드블록, ITP(추적 방지)로 인한 누락 10-30%
- **아임웹**: 취소/환불 반영 지연 (취소 시 매출 차감은 익일 반영)
- **광고**: API 오류 시 재시도 (일부 데이터 누락 가능)

### 3. 채널 분류 한계
- **Direct**: 북마크, 직접 입력, 추적 불가 트래픽 (실제는 다른 채널일 수 있음)
- **Organic Search**: SEO 트래픽 (브랜드 검색 vs 비브랜드 검색 구분 불가)
- **Referral**: 외부 사이트 링크 (일부는 광고일 수 있음)
- **Unassigned**: 미분류 트래픽 (UTM 없는 광고 등)

### 4. 전환 기여 문제
- **마지막 클릭 기준**: GA4 기본은 마지막 클릭 채널에 전환 귀속
- **실제 여정**: 사용자는 여러 채널 거쳐 구매 (예: 인스타 광고 → 구글 검색 → 구매)
- **결과**: Meta 광고의 실제 기여가 과소평가될 수 있음

### 5. 픽셀 vs 정본 차이
- **Meta 픽셀**: 10-20% 누락 또는 중복 (애드블록, 브라우저 설정)
- **GA4 이벤트**: 설정 오류, 지연, 중복 가능성
- **아임웹**: 결제 완료 기준 = 유일한 정본
- **결론**: 매출/전환은 **무조건 아임웹 정본 기준**으로 판단

## 🎯 지표별 활용 가이드

### 매일 확인 권장 지표
1. **Revenue** (어제 매출)
2. **Orders** (어제 주문)
3. **Spend** (어제 광고비)
4. **ROAS** (광고 효율)

### 주간 점검 권장 지표
1. **Sessions/Users** (트래픽 트렌드)
2. **CTR** (광고 크리에이티브 효과)
3. **Bounce Rate** (사이트 품질)
4. **Conversion Rate** (전환 효율)

### 이상 징후 판단
| 증상 | 가능한 원인 | 조치 |
|------|------------|------|
| 트래픽 ↓ + 매출 ↓ | 광고 중단 또는 효과 감소 | 광고 상태/예산 확인 |
| 트래픽 ↑ + 매출 ↓ | 트래픽 품질 저하 또는 사이트 문제 | 랜딩 페이지 점검 |
| CTR ↓ | 크리에이티브 피로도 또는 타겟팅 문제 | 소재 교체/타겟 조정 |
| 이탈률 ↑ | 페이지 로딩/UX 문제 | 기술적 오류 확인 |
| ROAS ↓ | 경쟁 심화 또는 시즌 영향 | 입찰가/예산 조정 |

## 🆕 2026-04-10 업데이트 — 일자별 퍼널 전환 차트 추가

### 추가 배경
대표님 피드백 기준으로, 매출 같은 최종 지표만으로는 **어느 날짜에 중간 퍼널 전환이 실제로 발생했는지** 한눈에 보기 어렵다는 문제가 있었습니다.
그래서 일별 지표 탭 안에 **중간 단계 전환 발생일 전용 차트**를 별도로 추가했습니다.

### 반영 위치
- **마케팅 워크플로우 대시보드 > 분석/리포트 > 일별 지표 탭**
- 기존 `트래픽 + 전환 + 광고비` 카드 바로 아래
- 카드명: **일자별 퍼널 전환**

### 현재 연결된 지표
1. **체험신청** — Supabase `feel_bookings` 일별 건수
2. **결제시작** — GA4 `begin_checkout` 유저 수
3. **결제정보입력** — GA4 `add_payment_info` 유저 수
4. **유효 판매전환** — 아임웹 본체판매 주문 기준 일별 판매전환 건수
5. **본체판매** — 아임웹 정본 본체 판매 대수

### 차트 해석 의도
이 차트는 아래 같은 질문에 바로 답하기 위한 용도입니다.
- 어떤 날짜에 **체험신청은 있었는데 결제로 안 넘어갔는가**
- 어떤 날짜에 **결제시작은 있었는데 결제정보입력에서 끊겼는가**
- 어떤 날짜에 **결제시작/결제정보입력은 있었지만 실제 본체판매는 없었는가**
- 어떤 날짜에 **중간 퍼널이 살아 있었는지 / 죽었는지**

즉, **최종 매출 차트의 보조지표**가 아니라, 일자별 병목 확인용 메인 차트로 쓰는 것이 목적입니다.
