# 마케터 온보딩 가이드

> 마케팅 워크플로우 대시보드 사용법
> URL: https://marketing-workflow-dashboard.pages.dev

---

## 🎯 이 도구가 뭔가요?

마케팅 실험을 **가설 기반으로 관리**하는 도구입니다.

**Before:** 대표님이 발견 → 바로 실행 → 결과 모름 → 다음 실행
**After:** 데이터에서 시그널 발견 → 가설 세우기 → 실행 → 결과 판정 → 학습 누적

---

## 📱 탭 구조

| 탭 | 용도 | 언제 쓰나 |
|:---|:---|:---|
| 📋 워크플로우 | 모든 실험 한눈에 | 매일 — 진행 상황 확인 |
| 📊 주간 리포트 | AI가 만든 주간 성과 분석 | 월요일 — 이번 주 데이터 확인 |
| 💡 시그널 | 발견한 이상 징후/기회 모아보기 | 리포트 본 후 — "이거 이상한데?" |
| 📚 Learn | 끝난 실험에서 배운 것 | 실험 종료 후 — 다음에 참고 |
| 🏷️ 소재 태깅 | 기존 광고 소재 태그 확인/수정 | 필요 시 — 소재 기획 참고 |
| ⚙️ 더보기 | 설계 배경, 분석 자료 | 참고용 |

## 🔄 주간 루틴

### 월요일: 리포트 확인 + 시그널 등록
1. **📊 주간 리포트 탭** 열기
2. 최신 리포트 읽기 — 특이사항, 전환 건별 상세, 관계 분석
3. **"이거 이상한데?"** 싶은 것 발견
4. **💡 시그널 등록** 버튼 클릭 → 등록

### 화~수요일: 설계서 작성
1. 시그널 중 실험할 것 선택
2. **+ 설계서 작성** 버튼 클릭
3. 가설 + 검증 기준 채우기 (필수!)
4. 대표님 승인 받기

### 목~금요일: 실행 + 결과 기록
1. 설계서대로 제작/집행
2. 상태를 "제작중" → "Live"로 변경
3. 검증 기간 후 결과 기록 → "완료"

## 💡 시그널 등록하는 법

### 방법 1: 에이전트 분석 (추천)
1. 시그널 등록 모달 상단에 **자유롭게 한 줄 적기**
   - 예: "이번주 메타 CPC가 179원까지 올라갔어"
   - 예: "카카오톡에서 배송 문의가 갑자기 늘었다"
2. **🤖 에이전트 분석** 클릭
3. 10~20초 후 아래 폼이 자동으로 채워짐
4. 내용 확인 → 수정할 거 수정 → **등록**

### 방법 2: 직접 입력
1. "또는 직접 입력" 아래 폼에 채우기
2. **제목** + **관찰 내용** + **데이터 출처** (필수)
3. 시그널 성격, 구체적 출처는 선택

### 데이터 출처 14가지
주간리포트 / Meta Ads / GA4 / 네이버SA / 아임웹주문 / CS카카오 / CS아임웹 / CS커뮤니티 / 인스타그램 / 현장 / 외부시장 / 내부논의 / 이전실험 / 기타

## 📝 설계서 작성하는 법

### 필수 입력
- **제목:** 무엇을 실험하는가
- **시그널:** 왜 이걸 하려고 하는가 (시그널 연결)

### 단계 게이트 (중요!)
**설계완료(design) 단계로 넘기려면** 아래 2개 필수:
- **가설:** "~하면 ~할 것이다" (예: "후기 영상 추가하면 커뮤니티 체류시간 2배 될 것이다")
- **검증 기준:** "N주 후 ~가 ~이면 성공" (예: "2주 후 체류시간 120초+ 달성")

가설과 검증 기준 없이는 다음 단계로 못 갑니다.

## 📊 리포트 읽는 법

### 구조 (4파트)
1. **🔔 이번 주 특이사항** — 가장 중요한 것 먼저 (두괄식)
2. **🛒 전환 건별 상세** — 이번 주 실제 구매가 있었다면 상세
3. **Part 1: 관계 분석** — "우리가 한 것(액션)과 수치가 어떻게 연결되는가"
   - A: 액션→지표 매칭됨
   - B: 액션했는데 지표 안 움직임
   - C: 외부 요인
   - D: 숨은 상관/시차
4. **Part 2: 지표 현황** — 채널별/퍼널/Meta/네이버/페이지 수치

### 시그널 찾는 팁
- **급변한 수치** — CPC가 갑자기 올랐다, CTR이 떨어졌다
- **기대와 다른 결과** — 소재 바꿨는데 성과 안 변했다 (케이스 B)
- **반복 패턴** — 매주 같은 문제가 나온다
- **새로운 발견** — 이전에 없던 채널에서 유입이 생겼다

## 📐 데이터 신뢰도 (방법론)

리포트와 분석에 🟢/🟡/🔴 표시가 있습니다:

| 표시 | 의미 | 예시 |
|:---:|:---|:---|
| 🟢 Tier 1 | **확정 팩트** — 데이터 직접 확인 | 주문 13건, 결제금액 ₩3.48M |
| 🟡 Tier 2 | **높은 신뢰 추론** — 교차 검증 가능 | 첫 방문→결제 "최소 N일" |
| 🔴 Tier 3 | **가설** — 검증 필요 | "광고 보고 나중에 검색으로 구매" |

### 주의할 표현
- ❌ "재방문자 = 구매자" → ✅ "결제 완료자의 63%가 이전 방문 이력 보유"
- ❌ "당일 구매 48%" → ✅ "GA4 쿠키 기준 (실제는 더 낮을 가능성)"
- ❌ "이 사람의 53일 여정" → ✅ "이 전환자의 첫 방문은 최소 53일 전"

## ⚡ 자주 묻는 질문

**Q: 시그널이 뭔가요?**
A: "이거 이상한데?" 또는 "이거 기회인데?" 하는 관찰입니다. 거창할 필요 없어요. "CPC가 올라가고 있다" 정도면 충분합니다.

**Q: 모든 시그널을 실험해야 하나요?**
A: 아니요. 시그널은 많이 등록하고, 그중에서 실험할 가치가 있는 것만 설계서로 발전시킵니다.

**Q: 가설을 어떻게 세우나요?**
A: "~하면 ~할 것이다" 형식. 예: "VP8(호기심) 소재를 늘리면 CTR이 5% 이상 나올 것이다." 틀려도 괜찮아요. 틀린 것도 학습입니다.

**Q: 에이전트 분석이 뭔가요?**
A: AI가 우리 데이터(CS 이력, 주간 리포트, 크리에이티브 스키마)를 참고해서 시그널을 분석해줍니다. 참고용이지 최종 판단은 사람이 합니다.

**Q: API 키가 뭐예요?**
A: 데이터를 저장하려면 인증 키가 필요합니다. 처음 접속 시 물어봅니다. 대표님이나 AI팀에게 받으세요.
