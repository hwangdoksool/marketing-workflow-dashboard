독술이 2026년 2~3월 본체 주문 13건을 대상으로 구매자 심층 분석 v2를 작성.
같은 날 GA4 × 아임웹 분석 방법론이 정립됨 (독술 발견, 시행착오 기반).

## 🔍 리뷰에서 발견된 문제 (5건)

### 1. 결제 이탈률 표 — 개인 데이터 오해 유발 ⚠️
- **문제:** 표 헤더가 "구매자 | 결제 시도 | 결제 완료 | 이탈률"로 되어 있어, 해당 수치가 그 구매자 개인의 결제 시도 횟수처럼 보임
- **실제:** 해당 날짜의 **사이트 전체** 결제 페이지 방문 수 (개인 특정 불가)
- **수정:** 표 제목을 "구매 발생일별 결제 퍼널 (사이트 전체 기준)"으로 변경. 구매자 이름 → "본체 주문 N건"으로 교체. 경고 문구를 표 위로 이동.
- **방법론 위반:** Tier 3 데이터를 Tier 1처럼 표시 (개인 매칭 불가 데이터를 개인별 표로)

### 2. "당일 ~48%" 표기 — 쿠키 한계 미강조 ⚠️
- **문제:** 카드에 "~48% — GA4 기준 첫 방문일에 결제"만 표시. 하단 해석에 단서가 있지만 카드만 보면 "절반이 즉시 구매"로 읽힘
- **실제:** 쿠키 기반이라 다른 기기 사전방문은 안 잡힘 → 실제 즉시구매는 48%보다 낮을 가능성
- **수정:** 카드 설명을 "GA4 쿠키 기준 (실제 즉시구매는 이보다 낮을 가능성 높음)"으로 변경
- **방법론 원칙:** 하한값은 항상 하한값임을 즉시 인지할 수 있어야 함

### 3. Meta display "리타겟팅" 암시 ⚠️
- **문제:** "메타 디스플레이 광고로 재유입 (리타겟팅 여부는 Meta에서 확인 필요)" — 괄호 안 단서가 있지만, "재유입"이라는 단어 자체가 리타겟팅을 연상
- **수정:** "리타겟팅인지 일반 노출 재클릭인지 Meta Ads Manager 확인 필요"로 구체화
- **방법론 원칙:** 금지 표현 — "리타겟팅" (미확인 시) → "재방문 유입"

### 4. 이준한 "219일 잠복" — 방법론 명백 위반 🔴
- **문제:** 3/25 이후 채널 분석에서 "이준한(219일 잠복)"이라고 서술
- **실제:** 아임웹 가입일 2025-08-24는 **결제 시 자동 생성되는 계정 날짜**가 아닌, 이전에 만든 계정일 수 있으나 "잠복 기간"으로 해석하면 안 됨. GA4 쿠키 기준 당일 전환 → 쿠키 리셋 가능성
- **수정:** "219일 잠복" 삭제. "아임웹 가입일이지 첫 방문일이 아님. GA4 쿠키 기준 당일 전환이므로 쿠키 리셋 가능성 높음"으로 교체
- **방법론 위반:** member_code 가입일 ≠ 첫 방문일 (시행착오 5.1). 금지 표현 — "N일 잠복/여정" (개인 특정 불가)

### 5. 파일명 날짜 혼동
- **문제:** `buyer-analysis-2026-02-03.html` — 2월 3일 분석처럼 보임
- **실제:** 2026년 2~3월 기간 분석
- **수정 미실시** (URL 변경 시 기존 공유 링크 깨짐). 문서 내 제목으로 커버.

## ✅ 잘 된 것

1. **Tier 표기 체계적 적용** — 🟢/🟡/🔴 + 구체적 근거/한계 명시
2. **cohort 오류 완전 수정** — "개인 여정" 서술 삭제, "탐색 페이지/채널 전이 데이터 삭제" 명시
3. **"재방문자 = 구매자" 회피** — "결제 완료자의 63%가 이전 방문 이력 보유"
4. **가설 A/B/C 분리** — 근거/한계/검증 방법 3요소 완비
5. **N=13 표본 한계 반복 명시** — "경향 수준", "일반화 곤란" 등
6. **교차 검증 표현** — "트래픽 피크 ≠ 구매 피크" 등 직관 반하는 발견 강조

이 케이스는 **리포트 리뷰 프로세스**의 필요성을 입증:
1. 작성자(독술)가 방법론을 정립하고 적용해도, 일부 표현/레이아웃에서 오해 소지가 남음
